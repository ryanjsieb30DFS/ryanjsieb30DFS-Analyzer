# MLB Park References

Per-park markdown files cataloguing park factors + stacking context. Used to provide cross-slate context when reviewing or planning an MLB Classic slate.

## Convention

- One `.md` file per park, named `<park_slug>.md` (e.g. `coors_field.md`)
- Park data comes from the slate's articles and the user — saved here as a cumulative knowledge base
- Each file has: physical profile (dimensions, altitude, typical weather/wind patterns, run environment, HR factors by handedness), strategic notes (how stacks play here — accumulates date-stamped "Per-slate observation" lines via the post-autopsy review)
- A file created mid-slate from articles alone is marked `**UNVERIFIED — built from this slate's articles only**` until a post-autopsy review confirms it

## Adding a new park

The Pre-flight ritual creates a stub automatically when a slate features a park not yet here; the post-autopsy review then appends what the slate actually proved or disproved about the park.
